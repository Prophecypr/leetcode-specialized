# Sliding Window — 滑动窗口

## 模板（C / C++）

### 定长窗口
```c
int left = 0;
for (int right = 0; right < n; right++) {
    // 更新窗口状态（加入 right）
    window[arr[right]]++;
    
    if (right - left + 1 > k) {
        // 移除 left
        window[arr[left]]--;
        left++;
    }
    // 处理窗口结果
}
```

### 不定长窗口（求最长/最大）
```c
int left = 0;
for (int right = 0; right < n; right++) {
    window[arr[right]]++;
    
    while (窗口不满足条件) {
        window[arr[left]]--;
        left++;
    }
    // 此时窗口满足条件，更新答案（求最长）
    ans = max(ans, right - left + 1);
}
```

### 不定长窗口（求最短/最小）
```c
int left = 0;
for (int right = 0; right < n; right++) {
    window[arr[right]]++;
    
    while (窗口满足条件) {
        // 在 left 右移前更新答案（求最短）
        ans = min(ans, right - left + 1);
        window[arr[left]]--;
        left++;
    }
}
```

## 例题笔记

### 3. 无重复字符的最长子串 ⭐

> 不定长滑动窗口 · 基础 · 里程碑：首道 C++ 题 + 首道不定长题

**与定长窗口的本质区别：**
- 定长：窗口满 k 后同时移入移出，**固定大小**
- 不定长：只控制**合法性**，不控制大小，**动态伸缩**

**思路：**
- 维护 `[left, right]` 窗口，保证窗口内无重复字符
- `right` 每次右移加入新字符 `c`，若 `cnt[c] > 1` → **用 `while` 收缩 left** 直到 `cnt[c] == 1`
- 每次更新 `ans = max(ans, right - left + 1)`

**为什么用 `while` 不是 `if`：**
- 连续删除多个字符才能恢复条件（比如窗口内有两个 'a' 和一个 'b'）
- 只用 `if` 一次只能删一个，可能删完还有重复

**学到什么：**
- 不定长窗口的**核心模式**：先扩→缩到合法→更新答案
- `unordered_map` 做频率计数，C++ 标准写法
- `left` 指针本质上也是 `i - window + 1` 的延迟表现形式

**模板对照：**
```cpp
int left = 0, ans = 0;
unordered_map<char, int> cnt;
for (int right = 0; right < n; right++) {
    cnt[s[right]]++;
    while (cnt[s[right]] > 1) {         // 不满足条件时收缩
        cnt[s[left]]--;
        left++;
    }
    ans = max(ans, right - left + 1);   // 更新最长
}
```

### 3090. 每个字符最多出现两次的最长子字符串

> 不定长滑动窗口 · 求最长/最大（基础）· 3 的直接变体

**与 3 的唯一区别：**
- 3：`while (cnt[c] > 1)` — 不能重复
- 3090：`while (cnt[c] > 2)` — 最多 2 次

**学到什么：**
- 不定长窗口模板的通用性：**只改 `while` 条件**就能适配不同约束
- 系列题对比：3（不重复）→ 3090（最多 2 次）→ 后续更宽松的约束也是同理
- 如果把条件改成 `while (cnt[c] > k)`，就是"每个字符最多出现 k 次"的通解

```cpp
while (cnt[s[right]] > 2) {  // 条件从 >1 改成 >2
    cnt[s[left]]--;
    left++;
}
```

### 1493. 删掉一个元素以后全为 1 的最长子数组

> 不定长滑动窗口 · 求最长/最大（基础）· 0 的计数版

**与 3/3090 的区别：**
| 题 | 约束条件 | 统计对象 | 答案公式 |
|----|----------|----------|----------|
| 3 | 每种字符 ≤1 | `cnt[char]` | `right-left+1` |
| 3090 | 每种字符 ≤2 | `cnt[char]` | `right-left+1` |
| 1493 | 最多 1 个 0 | `cnt0` | **`right-left`** |

**为什么答案用 `right-left` 而不是 `+1`：**
- 题目要求**必须删除一个元素**
- 无论窗口内是 0 个 0（删 1）还是 1 个 0（删 0），删除后的子数组长度 = 窗口长度 - 1
- 所以 `ans = max(ans, right - left)`（等价于 `(right-left+1) - 1`）

**学到什么：**
- `1 - nums[i]` 将 0/1 转换为 1/0 → 直接累加得到 0 的个数，省掉 if
- 本题的 `while` 条件 `cnt0 > 1` 也是模板的"改条件"思想
- 数组题（而不是字符串题）的滑动窗口通常更简单，无需哈希表

```cpp
int cnt0 = 0, left = 0, ans = 0;
for (int right = 0; right < n; right++) {
    cnt0 += 1 - nums[right];
    while (cnt0 > 1) {
        cnt0 -= 1 - nums[left];
        left++;
    }
    ans = max(ans, right - left);
}
```

### 3634. 数组元素最小移除数

> 不定长滑动窗口 · 求最长/最大（基础）· 排序 + 窗口

**思路：**
- 先排序，排序后窗口内最大值为 `nums[right]`，最小值为 `nums[left]`
- 维护窗口满足 `nums[right] / nums[left] ≤ k`（即 `nums[left] * k ≥ nums[right]`）
- 找最长合法子数组，答案 = `n - longest`
- 核心思想：**最小移除数 = 总数 - 最长合法子数组**

**为什么对：**
- 移除的元素可以任意选择，不要求连续
- 保留的部分最后必然是一个满足条件的子数组
- 排序后窗口内连续，相当于"保留哪些元素"的问题

**学到什么：**
- **"最少移除"问题**的通用转化：`remove = n - longest_valid_subarray`
- 排序 + 滑动窗口：当问题不要求保持原顺序时，排序可以简化判定条件
- `1LL` 防止 `nums[left] * k` 溢出
- `ranges::sort`（C++20）是 C++ 中最简洁的排序写法

```cpp
ranges::sort(nums);
int left = 0, ans = 0;
for (int right = 0; right < n; right++) {
    while (1LL * nums[left] * k < nums[right]) left++;
    ans = max(ans, right - left + 1);
}
return n - ans;
```

### 1208. 尽可能使字符串相等

> 不定长滑动窗口 · 求最长/最大（基础）· 最直白模板题

本题是模板翻译：谁都能一眼看出 `sum ≤ maxCost` 是不动点条件，`while (sum > maxCost)` 是收缩操作。

**与前几题对比：**
| 题号 | 窗口约束 | 收缩条件 | 额外操作 |
|------|----------|----------|----------|
| 3 | 无重复字符 | `cnt[c] > 1` | 哈希表 |
| 3090 | 最多重复 2 次 | `cnt[c] > 2` | 哈希表 |
| 1493 | 最多 1 个 0 | `cnt0 > 1` | `1-nums[i]` |
| 3634 | 排序后比值 ≤ k | `nums[l]*k < nums[r]` | 先排序 |
| **1208** | **总成本 ≤ maxCost** | **`sum > maxCost`** | **一行 abs** |

**学到什么：**
- 当约束条件是一维数值（成本、预算、和）时，滑动窗口是最自然解法
- 本题是"一维约束"类不定长窗口的标准模板

```cpp
int sum = 0, left = 0, ans = 0;
for (int right = 0; right < n; right++) {
    sum += cost(right);                     // 入窗
    while (sum > maxCost) {
        sum -= cost(left);                  // 收缩
        left++;
    }
    ans = max(ans, right - left + 1);       // 更新
}
```

### 904. 水果成篮

> 不定长滑动窗口 · 求最长/最大（基础）· 种类约束

**思路：**
本质是求最长子数组，其中**不同数字的种类数 ≤ 2**。

**与 3/3090/1493 的对比：**
| 题号 | 约束条件 | 判断方式 | 收缩操作 |
|------|----------|----------|----------|
| 3 | 每种字符 ≤1 | `cnt[c] > 1` | 只减计数 |
| 3090 | 每种字符 ≤2 | `cnt[c] > 2` | 只减计数 |
| 1493 | 最多 1 个 0 | `cnt0 > 1` | 只减计数 |
| **904** | **最多 2 种** | **`cnt.size() > 2`** | **减计数 + erase** |

**C++ 细节（你的学习记录）：**
- `cnt.size()` — unordered_map 当前有多少种 key（本题中即水果种类数）
- `cnt.erase(out)` — 当某种水果数量归零时，从 map 中移除该键
  - 好处：`cnt.size()` 自动反映真实种类数，无需额外维护 `sum/distinct` 变量
- `string` 支持 `.length()` 和 `.size()`；`vector<int>` 只有 `.size()`

```cpp
int left = 0, ans = 0;
unordered_map<int, int> cnt;
for (int right = 0; right < fruits.size(); right++) {
    cnt[fruits[right]]++;
    while (cnt.size() > 2) {
        int out = fruits[left];
        if (--cnt[out] == 0) cnt.erase(out);
        left++;
    }
    ans = max(ans, right - left + 1);
}
```

### 1695. 删除子数组的最大得分

> 不定长滑动窗口 · 求最长/最大（基础）· 求和版 3

**与 3 的关系：**
| 题号 | 维护量 | 求最值 | 数据结构 |
|------|--------|--------|----------|
| 3 | 窗口长度 | 最长 | unordered_map |
| **1695** | **窗口和** | **最大和** | **unordered_set** |

**用 `unordered_set` 代替 `unordered_map`：**
- 本题只关心元素是否**出现过**，不关心出现**几次**
- `set.contains(x)`（C++20）比 `map[key] > 0` 更语义化
- 去重时：`st.erase(nums[left])` 即可，无需检查计数归零

**学到什么：**
- 当只需存在性判断时，`set` 比 `map` 更轻量
- `contains()` 是 C++20 的语法，之前用 `st.count(x) > 0` 或 `st.find(x) != st.end()`
- 本题是 3 的"求和变体"——把 `right - left + 1` 换成 `sum`，约束条件完全相同

```cpp
unordered_set<int> st;
int ans = 0, sum = 0, left = 0;
for (int x : nums) {
    while (st.contains(x)) {
        st.erase(nums[left]);
        sum -= nums[left];
        left++;
    }
    st.insert(x);
    sum += x;
    ans = max(ans, sum);
}
```

### 1456. 定长子串中元音的最大数目

> 定长滑动窗口 · 基础题

**思路拆解：**
1. 右指针 `i` 遍历字符串，新字符是元音则 `vowel++`
2. 窗口未满 `k` 时（`i - k + 1 < 0`）继续扩大，不更新答案
3. 窗口满 `k` 后更新 `ans = max(ans, vowel)`
4. 若 `ans == k`（已达理论最大值），直接 `break` 早停
5. 滑出左边界字符：若是元音则 `vowel--`

**学到什么：**
- 定长窗口的典型写法：`left = i - k + 1` 定位左边界
- 元音判断条件长冗余 → 可考虑用位掩码或 `strchr("aeiou", c)` 简化
- **早停优化**：当 ans 达到 k 时不可能更大，直接返回，常数级优化但在某些 case 有效
- 先更新答案再滑出左边界（先判断后移除，顺序不能反）

**模板对照：**
```c
// 与本 repo 模板的对应关系
int left = 0;
for (int right = 0; right < n; right++) {
    if (isVowel(s[right])) vowel++;
    
    if (right - left + 1 > k) {
        if (isVowel(s[left])) vowel--;
        left++;
    }
    if (right - left + 1 == k) {
        更新答案;
    }
}
```

### 643. 子数组最大平均数 I

> 定长滑动窗口 · 基础题 · 踩坑记录 ⭐

**代码对比（第一次 → 第二次）：**

| 行 | 第一次（❌） | 第二次（✅） |
|----|------------|------------|
| 3  | `int ans=0,aver=0;` | `int ans=INT_MIN,aver=0;` |

**Bug 根因：ans 初始值选型错误**
- `ans=0` → 数组全为负数时，任何窗口和均为负，`MAX(0, 负数) = 0` → 返回错误
- `ans=INT_MIN` → 任何实际窗口和都 ≥ INT_MIN，正确

**延伸教训：**
- 求最大值时，初始值用 `INT_MIN`（或 `-inf`），不要想当然用 0
- 求最小值时，初始值用 `INT_MAX`
- 这个错误在滑动窗口、DP 中极其常见，属于**初始化惯性错误**
- 判断依据：**可能取值的下界/上界是什么**，而不是拍脑袋给 0

**次要问题：宏定义**
```c
#define MAX(a,b) (a>b) ? (a) : (b)   // ❌ 缺少外层括号
#define MAX(a,b) ((a)>(b) ? (a):(b)) // ✅
```
第一次的宏在 `ans=MAX(ans,aver)` 赋值场景下未触发优先级 bug（赋值优先级低于三元），但写成复杂表达式时（如 `MAX(a,b) + 1`）会出错。养成习惯：**宏定义整体和每个参数都加括号**。

**模板对照（与 1456 相同）：**
```c
// 唯一区别：初始值用 INT_MIN
int ans = INT_MIN, sum = 0;
for (int i = 0; i < n; i++) {
    sum += nums[i];
    if (i - k + 1 < 0) continue;
    ans = MAX(ans, sum);
    sum -= nums[i - k + 1];
}
return (double) ans / k;
```

### 1343. 大小为 K 且平均值大于等于阈值的子数组数目

> 定长滑动窗口 · 基础题 · 计数型

**与 1456/643 的区别：**
- 前两题是求最大/最优值，这题是**计数**
- 计数型不需要 `INT_MIN`/`INT_MAX`，`int ans=0` 即可

**学到什么：**
- **避免浮点运算**：`avg >= threshold` ↔ `sum >= threshold * k`
  - 用整数乘法替代浮点除法，性能更好且无精度问题
- 本题完全复用定长窗口模板，仅更新逻辑从 `ans = MAX(ans, ...)` 变为 `ans++`
- 窗口模板的通用性：同一种结构，改条件判断逻辑即可适配不同题目

```c
int ans = 0, sum = 0;
for (int i = 0; i < n; i++) {
    sum += arr[i];
    if (i - k + 1 < 0) continue;
    if (threshold * k <= sum) ans++;  // 唯一的不同
    sum -= arr[i - k + 1];
}
```

### 2090. K 半径子数组的平均值 ⭐ [难题，有门槛]

> 定长滑动窗口 · 基础 · 窗口中心与半径概念

**与前三题的本质区别：**
- 前三题窗口是连续的左→右，本题窗口是**以某点为中心，左右各 k**
- 需要返回**整个数组**（每个位置一个值），不是单个答案
- 边缘位置需要 fill -1（边界条件处理）

**核心技巧：滞后中心法**
```
右指针 i  → 窗口右端
中心位置  → i - k       （滞后 k 步）
左端位置  → i - 2k      （滞后 2k 步）
```

| 阶段 | i 范围 | left = i-2k | 做什么 |
|------|--------|-------------|--------|
| 窗口未满 | `[0, 2k-1]` | `< 0` | 只入不出 |
| 窗口饱满 | `[2k, n-1]` | `>= 0` | 计算中心 `i-k`，然后滑出 left |
| 遍历结束 | — | — | 剩余未赋值位置填 -1 |

**学到什么：**
- **带半径的定长窗口**本质还是定长窗口，只是计算目标位置时做了偏移
- **long long 防溢出**：k 最大 10^5，值最大 10^5，2k+1 个元素的和可能达到 `2*10^10`，超过 int32
- **两步填 -1**：先填左边缘 `[0, k-1]`，再填右边缘 `[n-k, n-1]`，中间由主循环填充

**模板对照：**
```c
int windowSize = 2 * k + 1;
long long sum = 0;
for (int i = 0; i < n; i++) {
    sum += nums[i];
    if (i - 2 * k < 0) continue;
    avgs[i - k] = sum / windowSize;   // 中心 = 右指针 - k
    sum -= nums[i - 2 * k];            // 左端 = 右指针 - 2k
}
```

### 2379. 得到 K 个黑块的最少涂色次数

> 定长滑动窗口 · 基础题 · 回归最简模板

**思路：**
- 窗口大小为 k，窗口内 `'W'` 的个数 = 需要涂色的次数
- 求所有窗口的最小值 → `INT_MAX` 初始化（643 的教训已固化 ✅）

**与 643 的对称性：**
| 题号 | 统计目标 | 求 | 初始值 |
|------|----------|---|--------|
| 643 | 窗口和 | 最大平均 | `INT_MIN` |
| 2379 | 窗口内 'W' 数 | 最小次数 | `INT_MAX` |

**学到什么：**
- 只维护一个计数器 `cntW` 即可，`cntB` 冗余
- **早停**：`ans == 0` 时直接返回（0 就是最优，不可能更小）
- 与 1456 结构完全对称：1456 统计元音求最大，2379 统计 W 求最小

```c
int ans = INT_MAX, cnt = 0;
for (int i = 0; s[i]; i++) {
    if (s[i] == 'W') cnt++;
    if (i - k + 1 < 0) continue;
    ans = MIN(ans, cnt);
    if (ans == 0) break;
    if (s[i - k + 1] == 'W') cnt--;
}
```

### 2461. 长度为 K 且元素互异的子数组最大和 ⭐

> 定长滑动窗口 · 基础 · 带频率去重

**相比前几题的新概念：**
- 前几题只维护窗口和/计数，此题额外需要**判断窗口内元素是否全部互异**
- 引入频率数组 `cnt[]` + 去重计数器 `distinct`
- 本质是**定长窗口 + 双条件**：长度 k ✅ + 元素互异 ✅

**频率跟踪模式：**
```
入窗时：cnt[x] 0→1 → distinct++
出窗时：cnt[x] 1→0 → distinct--
判断：   distinct == k → 窗口内全部互异
```

**学到什么：**
- 用 `calloc` 开计数数组（值范围有限时，比 hash map 更快更简单）
- `distinct` 计数器是频率去重题的通用技巧
- 与 2090 同为进阶题：2090 加半径约束，此题加互异约束，都是**在定长窗口基础上叠加条件**
- 本题是定长 → 变长时条件会变的过渡桥樑（2841 就是变长版）

**模板对照：**
```c
long long ans = 0, sum = 0;
int distinct = 0;
int* cnt = calloc(MAX_VAL + 1, sizeof(int));

for (int i = 0; i < n; i++) {
    sum += nums[i];
    if (cnt[nums[i]]++ == 0) distinct++;

    if (i - k + 1 < 0) continue;

    if (distinct == k) ans = MAX(ans, sum);

    int out = nums[i - k + 1];
    sum -= out;
    if (--cnt[out] == 0) distinct--;
}
free(cnt);
```

### 1423. 可获得的最大点数

> 定长滑动窗口 · 基础 · 反证法 ⭐[思维]

**常规思路（难）：** 从两端取 k 张，取法有 k+1 种，每次要算两端和，代码复杂。
**反证法（巧）：** 取完 k 张后，剩下的 `n-k` 张在中间**必然连续**。

```
max(取的) = total - min(剩下的中间连续子数组)
                  ↓
           定长滑动窗口，大小 = n-k，求最小和
```

**图解：**
```
卡片:  [1, 2, 3, 4, 5, 6, 1], k=3
取法:  左3张 (1+2+3) → 剩下 [4,5,6,1] 在中间连续
取法:  左2右1 (1+2+1) → 剩下 [3,4,5,6] 在中间连续
取法:  左1右2 (1+6+1) → 剩下 [2,3,4,5] 在中间连续
取法:  右3张 (1+6+5) → 剩下 [1,2,3,4] 在中间连续
                   ↑ 所有取法对应的中间段都是连续的
```

**学到什么：**
- **反向思维**：当正面复杂时，考虑补集（Complement）
- 将"两端取"转化为"中间剩"，把一个棘手问题化为标准定长窗口
- 这种"取=总-剩"的技巧在数组题中很常见
- 本题的窗口大小是**动态的**（取决于 k），但大小在循环内不变，仍属于定长窗口

**模板对照：**
```c
int window = n - k;                // 中间窗口大小
int minSum = INT_MAX, sum = 0;

for (int i = 0; i < n; i++) {
    sum += cardPoints[i];
    if (i - window + 1 < 0) continue;
    minSum = MIN(minSum, sum);
    sum -= cardPoints[i - window + 1];
}

return total - minSum;
```

### 1052. 爱生气的书店老板 ⭐

> 定长滑动窗口 · 进阶 · 双累加器分类

**核心创新：`s2[grumpy[i]]` 一行双分类**
```
s2[0] ← 不生气时的顾客（永远被服务，全程累加，不滑动）
s2[1] ← 生气时的顾客（仅在技巧窗口内被服务，需滑动求最大）
```

| 位置 | grumpy[i] | 计入哪里 | 是否滑动 |
|------|-----------|----------|----------|
| 不生气 | 0 | `s2[0]`（全量累加） | ❌ 固定值 |
| 生气且在窗口内 | 1 | `s2[1]`（窗内累加） | ✅ 随窗口滑动 |
| 生气且不在窗口 | 1 | 不计入任何 | — |

**学到什么：**
- 用 `bool/0/1` 做数组下标，省掉 if-else：`s2[grumpy[i]] += customers[i]`
- 双累加器思想：一个存"基线"，一个存"可优化量"
- 本题= 1423 的变体：都是"总-剩"的思维，不过这里总被拆成两路分别处理
- 出窗只用管 `grumpy=1` 的，`grumpy=0` 的出窗不影响 s2[1]（因为它本来就在 s2[0] 里永久保留）

```c
int s2[2] = {0};
for (int i = 0; i < n; i++) {
    s2[grumpy[i]] += customers[i];     // 分类累加
    if (i - minutes + 1 < 0) continue;
    ans = MAX(s2[1], ans);
    if (grumpy[i - minutes + 1]) s2[1] -= customers[i - minutes + 1];
}
return ans + s2[0];
```

## 常见类型

| 类型 | 特点 | 代表题 |
|------|------|--------|
| 定长 | 窗口大小固定 `right-left+1 == k` | 239. 滑动窗口最大值 |
| 不定长·最长 | 扩到不合法→缩到合法→更新 | **3. 无重复最长子串** |
| 不定长·最短 | 扩到合法→缩到不合法前更新 | 76. 最小覆盖子串 |
| 前缀和+窗口 | 用前缀和数组快速计算区间和 | 209. 长度最小子数组 |

## 易错点
- `while` 不是 `if`：窗口可能需要持续收缩
- 移除 left 时注意边界：先减再移，还是先移再减
- 字符串题注意 `strlen` 和数组下标的对应
- **`ans` 初始值**：求最大用 `INT_MIN`，求最小用 `INT_MAX`，不要惯性写 0（踩坑 643）
- **宏定义的括号**：整体和每个参数都要加括号 `#define MAX(a,b) ((a)>(b)?(a):(b))`
